*2000年二月十七日,的一天,死党BEN* 来Wolverhampton大厅接我.
我娇柔的胳膊下夹了个大纸板企鹅和一堆Linux光盘.
今天是一个特殊的日子: 微软 Windows 2000 发布.

就在几个星期前,亚利桑那州的世界另一边,
Deepak Saxena先生 [#]_ 出了个主意.
根据Windows 2000若隐若现的发布形势,
Deepak 希望看到Linux用户组和开源爱好者走上街头,
在微软这个重要里程碑的时间点上,
将同一个日期变成 "Linux 演示日",
发动数以千计的Linux支持者,传播他们挚爱的自由软件和开源思想.

仅仅几周之内,
Deepak 就说服了N多Lunix 领域的公司加入包括:
Caldera, EST, Linuxcare, Linux Central, Linux Mall, Red Hat, SGI, SuSE, Turbo Linux, 以及 TuxTops 都愿意掺合.
他从一些最大的在线技术新闻网站放出消息,
超过150个Linux用户组表示有感兴趣加入.
Deepak的小主意已成为一个全球协力的大工程!


我的小组就也是这个全球事件中的一员.
我们五个人钻进Ben的车,
用令人不安的速度飞奔到伯明翰(Birmingham)的附近.

我们对计算机和软件零售商,书店,图书馆进行走访,
并在某些情况下,直接在路上完全陌生的人们交谈.
相比我们,那些陌生人反而很不自在.

整整一天,
我们反复不断的解释为什么开放源码如此重要.
我们谈及校风,信仰,社会.
我们应用我们的热情消灭人们表现出来的困惑.
这是一个漫长的一天,
但是,当我们开车回我们的母校伍尔弗汉普顿(Wolverhampton)时,
我们为自个儿的努力而自豪.

我们的团结在英格兰的寒冷中带给彼此无尽的温暖.


Deepak的 *"Linux演示日"* 揭示了社会宣传的威力.
用令人信服的理由启动,
结合简单的工具和行为带动,
其结果是惊人的.
2000年2月17日的行动,其实对微软的影响是无关紧要的.
迷人的是其中的机制:
驱动全球的志愿者社区同一天完成同一行动的那种精神共享!

.. Deepak created ``buzz``. He generated interest, excitement, and a willingness to contribute to a shared campaign.

Deepak 创造了 ``buzz``. 他引发了兴趣/兴奋,并令人们愿意为一次分享行动作出贡献.



本章将剖析 ``buzz`` 这一学科.
Buzz是我们社会的一个重要组成部分,
它重要的是我们只有作对时才能获得它.
因此,我们将先探讨伟大 ``buzz`` 的关键特性,应该避免什么,
以及如何表达我们自己.

然后我们将继续前进讨论 ``buzz周期``
并针对不同的实际影响如何围绕你的项目创造 ``buzz`` .
